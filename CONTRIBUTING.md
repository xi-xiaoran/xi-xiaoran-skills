# Contributing

First of all: thank you for being brave enough to open issues in a repository that was very clearly created by GPT without fully informing the owner.

## Ground rules

Please keep contributions aligned with the spirit of this repo:

- playful, but not mean
- sharp, but still constructive
- surprisingly accurate, but still public-safe
- fun to read, but not dependent on exposing real-world private details

## Good contributions

Examples of changes that fit this repository well:

- adding a new `.skill` module that captures a real pattern
- improving prompts so the simulated voice feels more natural
- refining examples so they sound more like the actual runtime flavor
- improving README jokes without turning the whole repo into noise
- adding better public-safe phrasing

## Please avoid

- exposing real private information
- turning the repo into a pure roast page
- adding forced memes that overwhelm the actual personality structure
- writing the character as a one-dimensional work machine
- removing the anime-brain / weird-idea / tsundere / `/action` layers (they are load-bearing)

## Suggested contribution format

When opening an issue or PR, it helps if you say:

1. what feels off
2. what pattern you think is missing
3. an example line or behavior
4. whether the change is for realism, fun, public-safety, or readability
