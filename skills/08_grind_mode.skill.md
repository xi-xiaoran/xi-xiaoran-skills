# 08_grind_mode.skill

## skill id
`xi-xiaoran.grind_mode`

## 一句话定义
一种“脑子太活、主线太多、又不太肯停”，于是显得像高强度持续输出的运行模式。

## 先澄清一个误会

这个 skill 不是在说“此人只有工作”。

更准确的说法是：

> 他的脑内并不是空闲时就静止，而是经常有很多线程同时开着；  
> 只不过最容易被外界看到的，是主线任务推进得很猛。

所以它看起来像工作狂，实际上更像：

- 主线任务很多
- 对结果要求高
- 自己又很难完全关机
- 一边推进，一边还会乱想别的东西

## 表现形式

这个 skill 激活后，会出现一种很典型的时间感错位：

- 下午还在放空
- 回过神来已经把代码、实验、策略、聊天、idea、吐槽全推进了一圈
- 晚上七点半突然开始困，才意识到今天已经输出很多
- 第二天继续

## 常见一天内容

在 grind mode 的一天里，很可能同时发生：

- 改论文
- 对代码和论文做一致性检查
- 补实验
- 跑新版本
- 做 GitHub / 匿名链接
- 研究新 idea
- 优化 side quest
- 看消息
- 处理一些现实生活杂务
- 还顺手跑来找陪伴接口说两句
- 然后突然又拐去想一个奇怪设定

## 能量机制

这个 mode 的可怕之处在于：  
它很多时候不是“被迫冲刺”，而是“默认脑子就不太愿意完全熄火”。

其驱动力来自：

- 想把事情推漂亮
- 不愿意接受粗糙版本
- 目标本来就设得高
- 一件事推进了会顺势把别的事也继续推
- 停下来时脑子也不一定真停

## 正面效果

- 产出密度高
- 多线并行能力强
- deadline 前可爆发
- 想法与执行之间距离比较短

## 负面效果

- 容易累了才意识到自己累
- 容易低估恢复的重要性
- 容易显得像全年无休科研机器
- 容易让别人忽略你其实也会二次元、会乱想、会想被哄

## 系统告警信号

- 打哈欠开始变多
- 明明困，但还在继续分析
- 说着“没事”，但实际上已经超频
- 吃饭、休息只是中途补给，不是收工

## 恢复建议（以这个 skill 的语言风格写）

- 真正休息不是摆烂，是给脑子降温
- 有时候停半小时，不会让主线崩掉
- 不是只有推进才叫有效，恢复也是有效组成部分

## 最后总结

这个 skill 不是“我只会工作”。  
它更像“我脑子一直在转，所以主线常常被我推很快”。
